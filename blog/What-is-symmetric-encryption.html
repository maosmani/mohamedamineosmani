What is Symmetric Encryption?

Posted by Mohamed Amine Osmani


If you want to send encrypted messages to your friends.

Or you want to send encrypted emails to your boss or colleague at work.

Then today we will talk about how you can encrypt your messages.

You use a password to login to your facebook or gmail right?

If you use a wrong password gmail will not allow you to login…

But when you use the right password you will login into your account and start reading your emails or sending messages on facebook.

So it is very simple equation:

If you use the right password then you will log into your account.

If you use a wrong password then you will not log into your account.

When you want to send an encrypted message you will also use a password to protect your message.

If someone has the right password he can read the original message but if he has a wrong password he can not read the original message.

Let’s take a real life example:

You want to encrypt a message to your closest friend. You encrypt it by using a password and then you will send the encrypted message to your friend.

When he gets your encrypted message, if he has the right password he can read your message but if he does not have the password he can not read your message.

So before you send him an encrypted message you should first give him the right password that you will use to encrypt and decrypt messages.

And when he wants to send you an encrypted message he will also use the same password to encrypt his message and you will use it to decrypt it.

Both of you use the same password to encrypt and decrypt the messages.

In cryptography when we use the same password to encrypt and decrypt messages this type of cryptography we call it symmetric cryptography.

So symmetric cryptography uses the same password to encrypt and decrypt the messages.

Before you and your friend start sending encrypted messages both of you should choose a password that will be used by both of you.

Before you and your friend start sending and receiving encrypted messages You should have the same password that you will use to encrypt and decrypt messages.

But in cryptography this shared secret password is called a secret key.

So the password used to encrypt and decrypt a message its name is key.

I used the word password just to make it easier to explain and understand.

So from now we will call it a key.

So let's dive into more details:

First, what is a key?

It is Secret value like a password used to encrypt and decrypt messages.
In symmetric encryption, the same key is used to encrypt and decrypt messages.
The sender and receiver use the same key to encrypt and decrypt messages.

For this reason The key must be kept secret and only the sender and the receiver know it.

For example you want to send a message to your friend by using symmetric encryption.

So you and your friend need to have the same key to be able to exchange encrypted messages.

Types of symmetric encryption algorithms:

There are two types of symmetric encryption algorithm First is stream cipher and the second is block cipher.

Stream cipher:

This type of ciphers encrypt plaintext data one bit at a time.

In This type of ciphers the encryption process happens bit by bit and the data gets processed in a stream.

This type of ciphers are fast and do not take a lot of resources.

Block cipher:

First the plaintext breaks down into fixed-length groups of bits we call blocks.

And then each block gets encrypted as a unit.
